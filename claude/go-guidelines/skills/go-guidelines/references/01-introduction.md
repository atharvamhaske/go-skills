# 01 Introduction (Uber)

Use this guide as a code-quality and maintainability contract, not just formatting advice.

## Key intent

- Keep Go code manageable as teams and codebases grow.
- Prefer idioms that reduce ambiguity and maintenance burden.
- Keep conventions consistent across a package/repo.

## Practical defaults

- Use `goimports` for import formatting.
- Use static analysis and vet-like checks continuously.
- Apply rules pragmatically but consistently.
