# Contributing to go-skills

Thanks for helping improve this project.

## What to submit

- New Go skill packs
- Improvements to existing rules
- Additional references or automation scripts
- Better cross-harness compatibility (`SKILL.md`, `skills.md`, `AGENTS.md`)

## Submission flow

1. Fork this repository
2. Create a feature branch
3. Make focused commits
4. Open a Pull Request

## Pull Request checklist

- Keep rules clear and actionable
- Mention why the change is needed
- Add/update docs in the skill folder
- Keep changes scoped to related files

## Rule source policy

For `claude/go-guidelines/skills/go-guidelines`, keep guidance aligned to the Uber Go Style Guide unless explicitly introducing a new skill with a different source policy.
